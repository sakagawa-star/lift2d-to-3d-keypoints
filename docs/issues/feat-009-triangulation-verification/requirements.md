# feat-009: 三角測量による外部パラメータ検証 — 要求仕様書

## 目的

推定した外部パラメータ (R, t) の妥当性を、再投影誤差とは独立した手法で検証する。
2台のカメラの2D観測から三角測量で3D座標を復元し、既知の3D基準点座標との差を評価する。

## 背景

- 再投影誤差は1台のカメラ内で閉じた指標であり、系統的な誤差（Rとtの相補的なずれなど）を検出しにくい
- 三角測量は2台のカメラの推定結果を掛け合わせるため、一方の推定がおかしければ誤差が増幅されて顕在化する
- 既知の3D座標をGround Truthとして使えるため、復元精度を定量的に評価できる

## 入力

1. **推定結果TOML**: feat-008 の `--output` で出力されたTOMLファイル（全カメラの K, dist, R, t を含む）
2. **config.yaml**: `points_3d` と `points_2d` のパスを参照する（`target_camera` は使用しない）。パスは config.yaml の親ディレクトリからの相対パスとして解決する
3. **2D観測データ**: `points_2d_lab2.csv`（全カメラ分、config.yaml の `points_2d` で指定）
4. **3D基準点座標（Ground Truth）**: `kijunten_locations_lab2.csv`（config.yaml の `points_3d` で指定）

### 前提

- extrinsic.toml のセクション名（カメラ名）と points_2d.csv の `camera_name` カラムの値は完全一致する前提
- extrinsic.toml の `distortions` は4要素以上（OpenCV の distCoeffs として有効な長さ）を前提とする
- extrinsic.toml の `rotation` は3要素（Rodrigues ベクトル）、`translation` は3要素を前提とする

### 推定結果TOMLの形式

feat-008 の出力と同一:

```toml
[cam05520125]
name = "cam05520125"
size = [1920.0, 1080.0]
matrix = [[fx, 0.0, cx], [0.0, fy, cy], [0.0, 0.0, 1.0]]
distortions = [k1, k2, p1, p2, k3, k4, k5, k6]
rotation = [r1, r2, r3]
translation = [t1, t2, t3]
fisheye = false

[cam05520126]
...
```

## コマンドライン

```bash
uv run python verify_triangulation.py <config.yaml> <extrinsic.toml>
```

## 制約

- 共有基準点が2点未満のカメラペアはスキップする
- 三角測量には `cv2.triangulatePoints` を使用する
- 歪みの影響は三角測量前に `cv2.undistortPoints` で除去する
- `triangulatePoints` で同次座標の w 成分が極小（1e-6未満）の点は除外し警告を出す
- extrinsic.toml にカメラが2台未満の場合、エラーメッセージを出して終了する

## 品質基準

- 外部パラメータが正しければ、3D復元誤差は数cm以下になることが期待される（目安。シーンのスケールや基準点配置に依存する）
- 0.1m を超えるペアがあれば、どちらかのカメラの外部パラメータに問題がある可能性を示唆する（目安）
