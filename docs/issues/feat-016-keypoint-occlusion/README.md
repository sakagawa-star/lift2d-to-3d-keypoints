# feat-016: キーポイントのオクルージョン（深度による前後判定）

## Status: Open（feat-015 完了後に着手）

## 概要

feat-015（ピンホール3DGSレンダリング）の上に、人体3Dキーポイント（C3D, Halpe26）を投影し、**3DGSとの深度比較で前後関係（オクルージョン）を反映して**点＋ボーンで重ね描きする。カメラ手前の3DGS（障害物）に隠れるキーポイントは隠蔽する。

## 前提

feat-015 の完了（ピンホール3DGSレンダリング）。さらに本機能では gsplat の深度マップ（`render_mode="RGB+ED"`）を取得し、各キーポイントのカメラ深度と比較する。

## 検証で確立済みの事実（本機能の土台）

- **オクルージョンは技術的に可能**（実機確認済み）: gsplat 1.5.3 の `render_mode="RGB+ED"` で各画素の期待z深度が `(1,H,W,4)` の末尾チャンネルに、累積不透明度が `alphas`(1,H,W,1) に返る。深度はカメラ光軸方向のz深度で、キーポイントのカメラ深度 `(R·X+t)_z` と同じ単位で直接比較できる
- **floater対策が前提**: `near_plane=0.01` だと至近floaterで深度が汚染される（median 0.028m）。`near_plane=0.5` でfloaterを除去すると深度 median 1.944m（妥当な部屋の深度）になり、人体（1〜2m）と正しく比較できる
- **C3D→キャリブ座標変換**: 生座標 `(px,py,pz)` → `(pz, px, py)×0.001`。キーポイントがGT病室のベッド上に正確に乗ることで検証済み

## 範囲（予定。詳細は feat-015 完了後に requirements.md / design.md で確定）

- C3D読み込み（Halpe26抽出、residualによる欠損判定）
- キーポイント投影（歪みなし、カメラ深度算出）
- 深度マップによる可視判定（alpha閾値・深度マージン）
- 点＋ボーンの描画（オクルージョン考慮。ボーンは部分隠蔽）
- 連番PNG/MP4出力（複数フレーム）
