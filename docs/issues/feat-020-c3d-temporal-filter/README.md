# feat-020: C3Dキーポイントの時間方向平滑化スクリプト

## ステータス

Closed（2026-07-02 完了。手動テスト時の「90度回転」報告は取り込み手順起因と確定 → investigation.md）

## 概要

リフトアップ推定由来の3DキーポイントC3Dはフレーム間ジッター（"カクカク"）を含む。
C3Dファイル全体を時間方向にローパスフィルタ（Butterworthゼロ位相）で平滑化し、
新しいC3Dファイルとして書き出す独立スクリプト `phase4/filter_c3d.py` を新規作成する。

平滑化済みC3Dは以下の下流すべてで利用する:

- Blender への io_anim_c3d 取り込み（→ `fps_camera_pose.py` の入力アーマチュア）
- `render_keypoints.py` のキーポイント重ね描き

## 背景

- 対象データはベッド上に座位の人物。真の動きは低周波・小振幅で、推定ジッター（高周波）と
  周波数帯が明確に分離しているため、ローパスフィルタが有効
- `fps_camera_pose.py` 内部でのフィルタリング案は却下（カメラだけ滑らかになり、
  映像内の人体のカクつきが残る）。C3D→C3D の独立前処理スクリプトとして実装する
- 平滑化済みC3DのBlender再インポートは手作業で行う（ユーザー了承済み）
