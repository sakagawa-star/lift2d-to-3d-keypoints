# feat-020 要求仕様書: C3Dキーポイントの時間方向平滑化スクリプト

## 1.1 プロジェクト概要

- **何を作るのか**: C3Dファイル内の全マーカー軌跡を時間方向にローパスフィルタで平滑化し、
  新しいC3Dファイルとして書き出すコマンドラインスクリプト `phase4/filter_c3d.py` を新規作成する。
- **なぜ作るのか**: 2D→3Dリフトアップで推定したキーポイント（`npz_to_c3d.py` で変換したC3D）は
  フレームごとに独立な推定ノイズ（ジッター）を含み、Blender取り込み後のアーマチュアの動き・
  FPS頭部追従カメラの視線・`render_keypoints.py` の重ね描きがすべて"カクカク"する。
  C3D自体を平滑化することで、C3Dを消費するすべての下流を一括で滑らかにする。
- **誰が使うのか**: 本プロジェクトの開発者（リフトアップ済みキーポイントをBlender/レンダリングで
  利用するユーザー）。
- **どこで使うのか**: Linux（gtune2）、phase4 の uv 環境（`uv run --project phase4`）。
  Blender は不要（通常のPython環境で完結する）。

## 1.2 用語定義

| 用語 | 定義 |
|---|---|
| C3D raw座標 | C3Dファイルに格納されたマーカー座標。単位 mm。本プロジェクトでは world (X,Y,Z)m を (Y,Z,X)×1000 mm に入れ替えた値（feat-018） |
| マーカー | C3Dの point。本プロジェクトでは Halpe26 の関節点 |
| residual | C3D pointの4列目。負値はそのフレームのそのマーカーが無効（欠損）であることを示す |
| 有効サンプル | residual >= 0 かつ 座標3成分すべてが有限（NaN/Infでない）のサンプル |
| 無効サンプル | 有効サンプルでないサンプル |
| ゼロ位相フィルタ | 順方向・逆方向に2回フィルタを適用して位相遅れを打ち消すフィルタ（`scipy.signal.filtfilt`） |
| カットオフ周波数 | ローパスフィルタの遮断周波数 [Hz]。これより高い周波数成分を減衰させる |
| point rate | C3Dヘッダのフレームレート [Hz] |
| ギャップ | あるマーカーの時系列中で、有効サンプルに挟まれた無効サンプルの連続区間 |
| セグメント | あるマーカーの時系列を、長さが最大補間ギャップ長を超えるギャップで区切った連続区間。フィルタはセグメント単位で適用する |
| 最大補間ギャップ長 | 線形補間で埋めてよいギャップの最大フレーム数（`--max-gap`、デフォルト 10）。これを超えるギャップは補間せずセグメント境界とする |

機能設計書・コード内でも同じ用語を使う。

## 1.3 機能要求一覧

### FR-001: C3D読み込み

- **概要**: 入力C3Dファイルから全フレームのマーカー座標・residual・メタデータを読み込む。
- **入力**: C3Dファイルパス（位置引数）。
- **出力**: メモリ上の (F, J, 3) 座標配列 [mm]、(F, J) residual配列、マーカーラベル、
  point rate [Hz]、開始フレーム番号。
- **受け入れ基準**: `npz_to_c3d.py` が出力したC3Dを読み込み、フレーム数・関節数・ラベル・
  rate が py-c3d Reader の値と一致すること。

### FR-002: Butterworthゼロ位相ローパスフィルタによる平滑化

- **概要**: 各マーカー・各座標成分（X, Y, Z）の時系列に対し、2次Butterworthローパスフィルタを
  `scipy.signal.filtfilt` でゼロ位相適用する（実効4次・位相遅れなし）。
- **入力**: (F,) の時系列（マーカー×成分ごと）、カットオフ周波数 [Hz]（`--cutoff`、
  デフォルト 6.0）、サンプリング周波数 = C3D point rate [Hz]。
- **出力**: 平滑化された同じ長さの時系列。
- **受け入れ基準**:
  - 一定値の時系列を入力すると出力が同じ一定値になること（許容誤差 1e-6 mm）。
  - 低周波正弦波（カットオフの1/10の周波数）はほぼ保存され（振幅誤差5%以内）、
    高周波正弦波（カットオフの5倍の周波数、ただしNyquist未満）は振幅が10%未満に減衰すること。
  - 出力の位相遅れがゼロであること（低周波正弦波のピーク位置がずれないこと。許容1フレーム）。

### FR-003: 無効サンプル（欠損）の取り扱い

- **概要**: フィルタは連続した時系列を前提とするため、無効サンプルの扱いを以下と定める。
  長さが最大補間ギャップ長（`--max-gap`、デフォルト 10 フレーム）以下のギャップは
  線形補間で埋めてフィルタ計算に使う。それを超えるギャップは補間せず、時系列を
  ギャップの前後でセグメントに分割し、セグメントごとに独立にフィルタする
  （長い欠損を人工的な直線でつなぐと、その値がフィルタを介して周辺の有効サンプルに
  混入し軌跡を壊すため）。出力では元の無効サンプル位置を無効（residual = -1.0、
  座標 0.0）のまま維持する。補間値はフィルタ計算のためだけに使い、出力C3Dに
  有効値として書き出さない。
- **入力**: (F,3) 座標と (F,) residual（マーカーごと）、最大補間ギャップ長。
- **出力**: 有効サンプル位置のみ平滑化された座標。無効サンプル位置は入力と同じ無効表現。
- **受け入れ基準**:
  - 全サンプル有効のマーカーは全フレーム平滑化されること。
  - 最大補間ギャップ長以下の中間ギャップのあるマーカーで、有効位置が平滑化され、
    無効位置が出力でも無効のままであること。
  - 最大補間ギャップ長を超えるギャップでは補間が行われず、ギャップ前後のセグメントが
    それぞれ独立に平滑化されること（一方のセグメントの値が他方のフィルタ結果に
    影響しないこと）。
  - 先頭・末尾の無効区間（有効サンプルに挟まれていない区間）は補間せず、出力でも無効のままで、
    フィルタは各セグメント内にのみ適用されること。
  - セグメントの長さがフィルタ適用に必要な最小長（機能設計書で定義する
    `filtfilt` の既定パディング長 + 1 = 10 サンプル）未満の場合、そのセグメントは
    平滑化せず入力値のまま出力し、警告メッセージを標準出力に表示すること。

### FR-004: C3D書き出し

- **概要**: 平滑化後の座標を新しいC3Dファイルに書き出す。ラベル・point rate・
  開始フレーム番号は入力C3Dの値を維持する。単位・スクリーン軸は本プロジェクト規約の
  固定値（mm / X_SCREEN=+Z / Y_SCREEN=+Y、feat-018 で確定）で書く。この前提を保証する
  ため、入力C3Dの単位が mm でない場合、およびスクリーン軸パラメータが欠落している
  場合または規約値と異なる場合はエラー終了する（FR-005。本スクリプトの対象は
  本プロジェクト規約のC3D、すなわち feat-018 `npz_to_c3d.py` の出力に限定する）。
- **入力**: 平滑化後の (F, J, 3) 座標 [mm]、(F, J) residual、メタデータ。
- **出力**: C3Dファイル。出力パスは `--output` で指定、省略時は
  `<入力パスの拡張子除去>_filtered.c3d`。
- **受け入れ基準**:
  - 出力C3Dを py-c3d で読み戻し、フレーム数・ラベルが入力と一致すること
    （書き出し内で読み戻し検証を行い、失敗時は出力ファイルを残さないこと）。
  - 出力C3Dが `render_keypoints.py` の `load_c3d_all_frames` と Blender io_anim_c3d の
    双方で読み込めること（前者は自動テスト、後者は手動テストで確認する）。
  - 一時ファイル経由の原子的書き出しであること（書き出し途中の異常終了で
    最終出力パスに不完全なファイルが残らないこと）。
  - 出力パスが入力パスと同一の場合はエラー終了すること（上書き防止）。

### FR-005: パラメータ検証とエラー終了

- **概要**: 不正な入力・パラメータを検出し、日本語エラーメッセージを stderr に出力して
  終了コード1で終了する。
- **入力**: コマンドライン引数、入力C3Dの内容。
- **出力**: stderr へのエラーメッセージ、終了コード。
- **受け入れ基準**: 以下の各条件でエラー終了すること。
  - 入力C3Dファイルが存在しない、またはC3Dとして読み込めない。
  - カットオフ周波数が 0 以下。
  - カットオフ周波数が Nyquist周波数（採用された rate / 2）以上。
  - point rate が取得できない（0以下・非数値）かつ `--rate` による指定もない。
  - point rate が有効に取得でき、かつ `--rate` も指定された（どちらを使うか曖昧に
    なるため併存を禁止する。`--rate` はC3Dのpoint rateが欠損している場合の補完専用）。
  - `--max-gap` が 0 以下。
  - フレーム数が 0。
  - 入力C3Dの単位（POINT:UNITS）が `mm` でない（前後空白は無視して比較）。
  - 入力C3Dのスクリーン軸パラメータ（POINT:X_SCREEN / POINT:Y_SCREEN）が存在しない、
    または `+Z` / `+Y` と異なる（前後空白は無視して比較。欠落を許すと規約外C3Dが
    通過し出力時に規約値が付与されて姿勢解釈が黙って変わるため、存在必須とする）。
  - 開始フレーム番号が 1〜65534 の範囲外（py-c3d 0.6.0 のヘッダ16bit制約。feat-018 参照）。
  - 出力パスの拡張子が `.c3d` でない。
  - 出力パスが入力パスと同一（シンボリックリンク経由で同一実体を指す場合も含む）。

### FR-006: 処理結果のログ出力

- **概要**: 処理内容を標準出力に日本語で表示する。
- **入力**: なし（処理過程の情報）。
- **出力**: 標準出力。以下を表示する:
  入力パス、フレーム数、関節数、point rate、カットオフ周波数、
  最大補間ギャップ長、マーカーごとの無効サンプル数（1件以上あるマーカーのみ）、
  平滑化をスキップしたマーカー名とセグメント（FR-003）、
  平滑化による座標変位の統計（全有効サンプルの変位ノルムの平均・最大 [mm]）、
  出力パス。
- **受け入れ基準**: 正常終了時に上記項目がすべて表示されること。

## 1.4 非機能要求

- **パフォーマンス**: フレーム数 10,000・関節数 26 のC3Dを 30秒以内に処理する
  （フィルタ処理はNumPy/SciPyのベクトル演算であり、実際は数秒以内を想定）。
- **対応環境**: Linux、Python 3.10（phase4 uv 環境）。GPU不要。Blender不要。
- **信頼性**: 入力C3Dは変更しない（読み取りのみ）。出力は原子的書き出しで、
  異常終了時に不完全なファイルを最終パスへ残さない。
- **セキュリティ**: 該当なし（ローカルファイル処理のみ）。

## 1.5 制約条件

- **使用必須ライブラリ**: NumPy、SciPy（`scipy.signal.butter`, `scipy.signal.filtfilt`）、
  py-c3d（`c3d>=0.6.0`、既存依存）。
- **SciPy は phase4 の依存に新規追加する**（`phase4/pyproject.toml`）。追加時は
  `docs/TECH_STACK.md` に用途・選定理由・バージョンを追記する。
- **使用禁止**: Blender API（bpy）。本スクリプトはBlender外の通常Python環境で動作すること。
- **ネットワーク**: 不要（オフライン動作）。
- **メモリ**: 全フレームをメモリに読み込む方式でよい
  （26関節×10,000フレーム×3成分×8バイト ≒ 6MB であり問題にならない）。

## 1.6 優先順位（MoSCoW）

| 要求ID | 優先度 |
|---|---|
| FR-001 C3D読み込み | Must |
| FR-002 Butterworthゼロ位相フィルタ | Must |
| FR-003 無効サンプルの取り扱い | Must |
| FR-004 C3D書き出し | Must |
| FR-005 パラメータ検証 | Must |
| FR-006 ログ出力 | Should |

**MVP**: FR-001〜FR-005。全サンプル有効のC3D（現行パイプラインの実データは
`npz_to_c3d.py` 出力で、NaNを含まない限り全点有効）を平滑化して書き出せること。

**Won't（今回やらないこと）**:

- フィルタ種類の選択オプション（Savitzky-Golay、移動平均）。Butterworthゼロ位相のみ。
- `fps_camera_pose.py` / `render_keypoints.py` 側の変更（入力C3Dを差し替えるだけで効く）。
- NPZ段階でのフィルタリング、`npz_to_c3d.py` へのオプション追加。
- Blenderへの再インポート自動化（手作業で行う）。
