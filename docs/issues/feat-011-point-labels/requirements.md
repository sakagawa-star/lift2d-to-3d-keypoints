# 要求仕様書: feat-011 基準点番号ラベル表示オプション

## 1. 目的

`visualize_points_2d.py` が出力するアノテーション画像で、各プロット点がどの基準点に対応するかを目視で判別できるようにする。

## 2. 機能要求

### 2.1 ラベル表示オプション

- コマンドラインオプション `--label` を追加する
- `--label` 指定時、各基準点マーカーの近傍に**番号のみ**のテキストラベルを描画する
- `--label` 未指定時の動作は現状と完全に同一であること（後方互換）

### 2.2 ラベル内容

- ラベルは ObjectName から抽出した数字部分とする
  - 例: `基準_01` → `01`、`基準_12` → `12`
- 抽出ルール: ObjectName 中の**最後に現れる連続した数字列**を採用する（先頭ゼロは保持する）
- ObjectName に数字が含まれない場合は ObjectName 全体をラベルとして使用する
  - ただし OpenCV `putText` は非ASCII文字を描画できないため、非ASCII文字を含む場合は描画結果が `?` 等になる可能性がある。この場合の特別対応は行わない（番号付き命名が前提のため）

### 2.3 描画仕様

- ラベルは点マーカー（半径8pxの赤円）と重ならない位置（点の右上方向にオフセット）に描画する
- 文字色はマーカー（赤・黄）と区別しやすい色とする
- 視認性のため縁取り（背景色での太描き＋前景色での細描き）を行う
- ラベルが画像外にはみ出す場合（画像の上下左右いずれの端でも）は、はみ出さない位置に調整（クランプ）する。ただし縁取り・アンチエイリアスの滲みによる数px程度のはみ出しは許容する（クランプはテキスト原点基準で行うため）

### 2.4 対象点

- 画像範囲内に描画される点のみラベルを付与する（範囲外スキップ点は現状どおりスキップ）

## 3. 非機能要求

- 追加の外部依存（Pillow 等）を導入しない。OpenCV のみで実現する
- 既存の出力ファイル名規則（`<stem>_annotated<suffix>`）は変更しない
- コメント・出力メッセージは日本語、関数シグネチャに型ヒントを付ける（コーディング規約準拠）

## 4. 入出力

### 入力

```bash
cd phase0
uv run python visualize_points_2d.py data/config.yaml --label
uv run python visualize_points_2d.py data/config.yaml --camera cam01 --label
```

config.yaml の要件は現状と同じ（`points_2d`、`image_<camera_name>` キー等）。

### 出力

- 現状と同じパスのアノテーション画像。`--label` 指定時のみ番号ラベル付き

## 5. 品質基準

- `--label` 未指定時、出力画像が現行実装と同一であること
- `--label` 指定時、画像範囲内の全点に番号が描画されること
- 数字を含まない ObjectName でもエラーにならないこと
- tests/ にユニットテストを追加し、`uv run pytest -v` が全て成功すること

## 6. 対象外（スコープ外）

- 日本語（非ASCII）ラベルの描画（Pillow 依存が必要なため対象外）
- ラベルの色・フォントサイズのコマンドラインからの変更
- estimate_camera_params.py 等、他スクリプトへの波及
- 重複番号の検出・警告（例: `右_基準_01` と `左_基準_01` がともに `01` になるケース。現データは `基準_NN` 形式のため対象外）
- 番号の後ろに別の数字列が付く命名（例: `基準_01_v2` は最後の数字列 `2` がラベルになる。現データでは発生しないため特別対応しない）
