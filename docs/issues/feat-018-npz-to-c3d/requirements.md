# feat-018 要求仕様書: NPZ→C3D変換スクリプト

## 1.1 プロジェクト概要

- **何を作るのか**: 2Dキーポイントを3Dにリフトアップした NPZ ファイルを、Blender の
  C3D インポートアドオン `io_anim_c3d` で取り込める C3D ファイルへ変換する CLI スクリプト
  `phase4/npz_to_c3d.py`。
- **なぜ作るのか**: リフトアップ済み3Dキーポイント（NPZ形式）を、既存の C3D ベース
  ワークフロー（Blender可視化、将来的には `render_keypoints.py` での重ね描き）に
  乗せるため。現状 NPZ は直接 Blender に取り込めない。
- **誰が使うのか**: 本プロジェクトの開発者（キャリブ・可視化検証を行う人）。
- **どこで使うのか**: phase4 の uv 環境（`phase4/.venv`、py-c3d 0.6.0 導入済み）。
  CLI から `uv run --project phase4 python npz_to_c3d.py ...` で実行する。

## 1.2 用語定義

機能設計書・コード内でも同じ用語を使う。

| 用語 | 定義 |
|---|---|
| NPZ | 入力ファイル。`numpy.savez` 形式。`phase4/data/session001_f145749_world300_spec.md` に仕様。本案件が読むキーは `x3d_world`, `frame_ids`, `joint_names`。 |
| world座標 | NPZ `x3d_world` の座標系。単位メートル、軸配列 `(X,Y,Z)`。本案件対象データでは実測により鉛直上方向は `+Z`（頭が +Z 側）。spec §4 本文の「y=垂直」は「←キャリブ依存」の注記どおりキャリブ依存であり、本データには適用しない。 |
| C3D raw 座標 | C3D ファイルに格納する生の3値 `(px,py,pz)`。本案件では単位 mm。 |
| `c3d_to_calib` | `render_keypoints.py` の既存関数。C3D raw `(px,py,pz)` mm → `(pz,px,py)×0.001` m。 |
| io_anim_c3d | Blender 用 C3D インポートアドオン（`/home/sakagawa/git/io_anim_c3d/`）。 |
| X_SCREEN/Y_SCREEN | C3D の `POINT:X_SCREEN`/`POINT:Y_SCREEN` パラメータ。io_anim_c3d が軸の向き解釈に使う。 |
| residual | C3D 各点の4列目。`>= 0` で有効サンプル、`< 0` で無効サンプル。 |

## 1.3 機能要求一覧

### FR-001: NPZ読み込みと検証

- **概要**: NPZ を読み込み、必要キーと形状・整合性を検証する。
- **入力**: NPZ ファイルパス（CLI 位置引数）。
- **出力**: メモリ上の `x3d_world` (F,22,3) float、`frame_ids` (F,) int、`joint_names` (22,) str。
- **受け入れ基準**:
  - `x3d_world`, `frame_ids`, `joint_names` のいずれかが欠ける場合、エラーメッセージを
    標準エラーへ出力し、終了コード 1 で終了する。
  - `x3d_world` が3次元 (F, J, 3) でない、または末尾次元が 3 でない場合、エラー終了する。
  - `frame_ids` の長さが `x3d_world` のフレーム数 F と一致しない場合、エラー終了する。
  - `joint_names` の長さが `x3d_world` の関節数 J と一致しない場合、エラー終了する。
  - `frame_ids` が昇順の連番（差分がすべて 1）でない場合、エラー終了する
    （C3D はフレーム番号が連続である前提のため）。

### FR-002: world座標 → C3D raw 座標 変換

- **概要**: world座標（m）を C3D raw 座標（mm）へ、軸入替を伴って変換する。
- **入力**: `x3d_world` (F,J,3) m。
- **出力**: C3D raw 座標 (F,J,3) mm。変換式は `raw = (Y, Z, X) × 1000`。
- **受け入れ基準**:
  - 出力 raw に `c3d_to_calib`（`(px,py,pz)→(pz,px,py)×0.001`）を適用すると、
    元の `x3d_world` と一致する（数値誤差 1e-4 m 以内）。
  - NaN/Inf を含む点は変換後も値を保持してよいが、FR-003 で無効サンプルとして書き出す。

### FR-003: C3D書き出し

- **概要**: 変換した座標を C3D ファイルに書き出す。io_anim_c3d が正しく取り込める
  メタデータを設定する。
- **入力**: FR-002 の raw 座標、`frame_ids`、`joint_names`、フレームレート（FR-004）。
- **出力**: C3D ファイル（パスは FR-004）。
- **受け入れ基準**:
  - `POINT:LABELS` に `joint_names` の J 個のラベルがそのままの順序・文字列で書き込まれる。
  - `POINT:UNITS` に `'mm'` が設定される。
  - フレームレート（`point_rate`）が FR-004 で決まる値（既定 30.0 Hz）で書き込まれる。
  - C3D のフレーム番号は **1 始まりの連番（1..F）** とする（`set_start_frame(1)`）。
    NPZ の絶対フレーム番号 `frame_ids`（例: 145599〜）は C3D のフレーム番号としては
    保持しない。理由: py-c3d 0.6.0 のヘッダ first/last frame は 16bit（65535 上限）で、
    65535 を超える開始フレーム（本案件の 145599）を渡すと読み戻しが破綻する
    （実測: `first_frame=145595`, `count=301`, EOF 警告）。
  - C3D フレーム番号 `i`（1..F）と NPZ 絶対フレーム番号 `frame_ids[i-1]` の対応関係を、
    実行時に標準出力へ表示する（先頭・末尾の対応と総数）。
  - `POINT:X_SCREEN='+Z'`, `POINT:Y_SCREEN='+Y'` が設定される（FR-005 の Blender 正立条件。
    io_anim_c3d は pose bone のローカル座標として描画するため、表示鉛直は `Y_SCREEN` で決まる。
    design.md「Blender 取り込み時の座標変換」参照）。
  - 各点の residual は、座標が有限なら `0.0`（有効）、NaN/Inf を含むなら `-1.0`（無効）。
  - `MANUFACTURER:SOFTWARE` を設定しない（io_anim_c3d のラベル間引きを発生させないため）。
  - 出力 C3D を py-c3d で読み戻すと、フレーム数 F・関節数 J・ラベル・各点座標
    （`c3d_to_calib` 適用後に `x3d_world` と 1e-4 m 以内で一致）が入力 NPZ と整合する。
    フレーム番号は 1..F の連番で読み戻せる。

### FR-004: CLIインターフェース

- **概要**: コマンドライン引数で入出力パスとフレームレートを指定する。
- **入力/出力（引数）**:
  - 位置引数 `npz_path`: 入力 NPZ パス（必須）。
  - `--output PATH`: 出力 C3D パス。省略時は入力 NPZ と同じディレクトリ・同じ basename で
    拡張子を `.c3d` にしたパス。
  - `--fps FLOAT`: フレームレート [Hz]。既定 30.0。
- **受け入れ基準**:
  - `npz_path` を渡さずに実行した場合、argparse の使用法エラーで終了する。
  - `--output` 省略時、`data/foo.npz` に対し `data/foo.c3d` が生成される。
  - 出力パスの拡張子が `.c3d`（大文字小文字無視）でない場合、エラーメッセージを stderr へ
    出力し終了コード 1 で終了する（誤って入力ファイルを指定する事故を防ぐ）。
  - 出力パスの絶対パスが入力 NPZ の絶対パスと一致する場合、エラー終了する
    （入力ファイルの上書き・破壊を防ぐ）。
  - 正常終了時は終了コード 0、出力パスを標準出力に表示する。

### FR-005: Blender io_anim_c3d での取り込み確認（受け入れ確認）

- **概要**: 生成した C3D が io_anim_c3d で正しく取り込め、人体が正立して見えること。
- **入力**: 生成 C3D。
- **出力**: Blender 上の armature（22ボーン）+ アニメーション。
- **受け入れ基準**:
  - io_anim_c3d の標準設定（`use_manual_orientation=False`）で取り込んだとき、
    全22ラベルがボーンとして生成される（ラベル間引きで消えない）。
  - 取り込み後、人体の垂直方向（world の鉛直軸 = Z。spec §4 本文は「y=垂直」だが
    「←キャリブ依存」の注記どおり、本データの鉛直軸は実測で Z）が Blender の上方向（+Z）に
    一致して表示される（横倒し・上下逆さまでない）。
  - スケールがメートルで取り込まれる（身長がおよそ実寸: 数値オーダーが 0.1〜2 m 程度）。

## 1.4 非機能要求

- **パフォーマンス**: 300フレーム×22関節規模で、変換・書き出しは 5 秒以内に完了する。
- **対応環境**: Linux + phase4 の uv 環境（Python 3.10、py-c3d 0.6.0）。GPU 不要。
- **信頼性**:
  - 入力検証に失敗した場合は出力ファイルを生成せずにエラー終了する（壊れた C3D を残さない）。
  - C3D 書き出しは同一ディレクトリの一時ファイルに行い、py-c3d での読み戻し検証
    （フレーム数 F・ラベル一致）に成功した場合のみ `os.replace` で最終パスへ確定する。
    検証失敗時は一時ファイルを削除し、最終パスを変更しない。

## 1.5 制約条件

- **使用必須ライブラリ**: NumPy、py-c3d（`import c3d`、phase4 venv に導入済み 0.6.0）。
- **新規ライブラリ追加禁止**: 既存 phase4 環境に無いライブラリを追加しない
  （追加が必要と判明した場合は中断して報告する）。
- **Blender 依存禁止**: 本スクリプトは Blender / bpy に依存しない（純粋な NumPy + py-c3d）。
- **ネットワーク**: オフラインで動作すること。
